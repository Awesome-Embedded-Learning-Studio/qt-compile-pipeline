# 配置指南

## 配置文件总览

本项目采用模块化的配置管理方式，所有配置文件存放在 `config/` 目录下。每个配置文件负责特定的配置领域，便于维护和理解。

### 配置文件列表

| 配置文件 | 用途说明 | 必需性 |
|---------|---------|--------|
| `qt.conf` | Qt 版本、源码地址、模块列表配置 | 必需 |
| `toolchain.conf` | 交叉编译工具链配置 | 交叉编译时必需 |
| `environment.conf` | 编译环境变量配置 | 可选 |
| `cmake.conf` | CMake 构建选项配置 | 可选 |

### 配置文件目录结构

```
config/
├── qt.conf              # Qt 版本与源码配置
├── toolchain.conf       # 工具链配置
├── environment.conf     # 环境变量配置
└── cmake.conf          # CMake 选项配置
```

## 配置加载顺序

脚本按以下顺序加载配置文件：

1. `config/qt.conf` - Qt 基础配置
2. `config/toolchain.conf` - 工具链配置（仅交叉编译时）
3. `config/environment.conf` - 环境变量配置
4. `config/cmake.conf` - CMake 选项配置

后加载的配置会覆盖前面的同名变量。

## 配置文件语法

所有配置文件均使用 Bash 语法，支持：

- 变量赋值：`VAR="value"`
- 变量引用：`${VAR}` 或 `$VAR`
- 命令替换：`$(command)`
- 注释：`# 注释内容`

### 配置验证

部分配置文件内置变量验证机制，使用 Bash 参数展开进行空值检查：

```bash
: "${VAR:?Error: VAR must be set and non-empty}"
```

当变量为空时，脚本会立即报错退出，防止后续操作产生意外结果。
