# 工具链配置

本节介绍交叉编译工具链的配置方法，包括配置项说明、常用工具链前缀列表以及验证工具链安装的方法。

## 配置文件

工具链配置位于 `config/toolchain.conf` 文件中。

```bash
# 加载工具链配置
source config/toolchain.conf
```

## 配置项说明

### TOOLCHAIN_URL

工具链压缩包的下载地址。

- **用途**：指定工具链的下载 URL
- **默认值**：`""`（空字符串，表示不下载）
- **行为**：
  - 非空时：自动下载并解压到 `TOOLCHAIN_ROOT` 目录
  - 为空时：直接使用本地 `TOOLCHAIN_ROOT` 目录的工具链

```bash
# 示例：从 Linaro 下载 ARM 工具链
TOOLCHAIN_URL="https://releases.linaro.org/components/toolchain/binaries/7.5-2019.12/arm-linux-gnueabihf/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf.tar.xz"

# 使用本地工具链（跳过下载）
TOOLCHAIN_URL=""
```

常用工具链下载源：
- **Linaro**：https://releases.linaro.org/components/toolchain/binaries/
- **ARM 官方**：https://developer.arm.com/downloads/-/gnu-rm
- **NXP**：https://www.nxp.com/design/software/embedded-software/

### TOOLCHAIN_ROOT

工具链的根目录路径。

- **用途**：指定工具链的安装位置
- **默认值**：`"/opt/arm-gnu-toolchain/"`
- **行为**：
  - `TOOLCHAIN_URL` 非空时：下载的工具链解压到此目录
  - `TOOLCHAIN_URL` 为空时：直接使用此目录作为工具链路径

```bash
# 使用系统默认位置
TOOLCHAIN_ROOT="/opt/arm-gnu-toolchain/"

# 使用自定义位置
TOOLCHAIN_ROOT="${HOME}/toolchains/arm-linux-gnueabihf/"
```

### TOOLCHAIN_PREFIX

编译器可执行文件的前缀。

- **用途**：指定交叉编译工具链的前缀
- **默认值**：`"arm-none-linux-gnueabihf-"`
- **必填**：是
- **说明**：编译器名称格式为 `{PREFIX}gcc`，如 `arm-linux-gnueabihf-gcc`

```bash
# ARM 32位
TOOLCHAIN_PREFIX="arm-linux-gnueabihf-"

# ARM 64位
TOOLCHAIN_PREFIX="aarch64-linux-gnu-"
```

### TARGET_ARCH

目标系统架构。

- **用途**：指定目标系统的处理器架构
- **默认值**：`"armhf"`
- **常用值**：`arm`、`armhf`、`aarch64`、`mips`、`x86_64`

```bash
# ARMv7 (32位)
TARGET_ARCH="armhf"

# ARMv8 (64位)
TARGET_ARCH="aarch64"
```

## 常用工具链前缀列表

### ARM 32位

| 前缀 | 说明 | 目标平台 |
|------|------|----------|
| `arm-linux-gnueabihf-` | ARM 硬浮点工具链 | ARMv7-A, ARMv8-A (32位) |
| `arm-none-linux-gnueabihf-` | ARM 无厂商标识工具链 | ARMv7-A |
| `arm-linux-gnueabi-` | ARM 软浮点工具链 | ARMv5TE, ARMv6 |
| `arm-none-eabi-` | ARM 嵌入式工具链 | Cortex-M/R 系列微控制器 |

### ARM 64位

| 前缀 | 说明 | 目标平台 |
|------|------|----------|
| `aarch64-linux-gnu-` | AArch64 GNU/Linux 工具链 | ARMv8-A (64位) |
| `aarch64-none-linux-gnu-` | AArch64 无厂商标识工具链 | ARMv8-A |
| `aarch64-none-elf-` | AArch64 嵌入式工具链 | ARMv8-M/R |

### 其他架构

| 前缀 | 说明 | 目标平台 |
|------|------|----------|
| `mips-linux-gnu-` | MIPS 32位 | MIPS32 |
| `mips64-linux-gnuabi64-` | MIPS 64位 | MIPS64 |
| `riscv64-linux-gnu-` | RISC-V 64位 | RISC-V |
| `powerpc-linux-gnu-` | PowerPC 32位 | PowerPC |
| `powerpc64le-linux-gnu-` | PowerPC 64位小端 | POWER8/9 |

## 验证工具链安装

### 1. 检查编译器版本

```bash
# 加载配置后检查编译器版本
source config/toolchain.conf
${TOOLCHAIN_BIN_DIR}/${TOOLCHAIN_PREFIX}gcc --version
```

输出示例：
```
arm-linux-gnueabihf-gcc (Ubuntu/Linaro 7.5.0-3ubuntu1~18.04) 7.5.0
Copyright (C) 2017 Free Software Foundation, Inc.
This is free software; see the source for copying conditions.  There is no such warranty; not even for MERCHANTABILITY or FITNESS FOR A PARTICULAR PURPOSE.
```

### 2. 验证工具链完整性

```bash
# 检查关键工具是否存在
for tool in gcc g++ ar ranlib strip; do
    which ${TOOLCHAIN_PREFIX}${tool} && echo "[OK] ${TOOLCHAIN_PREFIX}${tool}"
done
```

### 3. 测试编译简单程序

```bash
# 创建测试程序
cat > test.c << 'EOF'
#include <stdio.h>
int main() {
    printf("Hello from cross-compiler!\n");
    return 0;
}
EOF

# 编译测试
${TOOLCHAIN_PREFIX}gcc -o test test.c
file test
```

输出应该显示为 ARM 可执行文件：
```
test: ELF 32-bit LSB executable, ARM, EABI5 version 1 (SYSV)...
```

### 4. 使用自动化验证脚本

项目提供了工具链验证脚本：

```bash
./scripts/verify_toolchain.sh
```

## 使用本地工具链 vs 下载工具链

### 方式一：使用本地工具链

适用于已安装工具链的系统。

**配置：**
```bash
# config/toolchain.conf
TOOLCHAIN_URL=""                    # 留空，不下载
TOOLCHAIN_ROOT="/opt/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/"
TOOLCHAIN_PREFIX="arm-linux-gnueabihf-"
```

**优点：**
- 无需下载，构建速度快
- 可复用系统已安装的工具链
- 适合持续集成环境

### 方式二：自动下载工具链

适用于没有预装工具链的系统。

**配置：**
```bash
# config/toolchain.conf
TOOLCHAIN_URL="https://releases.linaro.org/components/toolchain/binaries/7.5-2019.12/arm-linux-gnueabihf/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf.tar.xz"
TOOLCHAIN_ROOT="${WORK_DIR}/toolchains/gcc-linaro-7.5.0/"
TOOLCHAIN_PREFIX="arm-linux-gnueabihf-"
```

**优点：**
- 自动化程度高，无需手动安装
- 构建环境可复现
- 适合 Docker 容器或临时构建环境

## 常见问题

### 工具链找不到

**问题**：`which ${TOOLCHAIN_PREFIX}gcc` 返回空

**解决方法**：
1. 检查 `TOOLCHAIN_BIN_DIR` 是否正确
2. 确保 `TOOLCHAIN_ROOT` 目录存在且包含工具链文件
3. 手动设置 `TOOLCHAIN_BIN_DIR`

```bash
# 手动指定 bin 目录
TOOLCHAIN_BIN_DIR="/opt/gcc-linaro-7.5.0/bin/"
```

### 权限问题

**问题**：无法执行工具链二进制文件

**解决方法**：
```bash
# 添加执行权限
chmod +x ${TOOLCHAIN_BIN_DIR}/*
```

### 库依赖问题

**问题**：工具链执行时提示找不到动态库

**解决方法**：
```bash
# 检查工具链依赖的库
ldd ${TOOLCHAIN_BIN_DIR}/${TOOLCHAIN_PREFIX}gcc

# 安装缺少的库（Ubuntu/Debian）
sudo apt-get install lib32stdc++6 lib32z1
```
