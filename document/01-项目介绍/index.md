# 项目介绍

## 项目定位

Qt6 交叉编译自动化管道（Qt6 Cross-Compilation Pipeline）是一个专门为嵌入式 Linux 开发设计的 Qt 编译工具链。本项目以 **third-party** 仓库的形式存在，可被其他项目直接引用，实现 Qt 的自动化构建。

**核心设计理念**：不造轮子，只是将 Qt 官方编译流程自动化、配置化、可复现化。

---

## 核心功能

### 1. 一键完成 Qt 全流程构建

```bash
./build.sh
```

单个命令即可完成：
- Qt 源码自动下载（支持断点续传）
- 依赖检测与自动安装
- CMake 配置与编译
- 安装与打包
- 校验和生成

### 2. Host Qt 与 Target Qt 交叉编译支持

- **Host Qt**：在开发机上运行的 Qt，用于开发阶段
- **Target Qt**：交叉编译到目标架构（ARM/ARM64）的 Qt，用于部署到嵌入式设备

两种 Qt 可独立构建，也可在一次构建流程中同时生成。

### 3. 配置驱动，极简定制

所有编译参数通过 `config/` 目录下的配置文件管理：

```
config/
├── qt_host.conf      # Host Qt 配置
├── qt_target.conf    # Target Qt 配置
└── toolchain.cmake   # 交叉编译工具链配置
```

只需修改配置文件，无需修改脚本，即可实现：
- Qt 版本切换（6.5.x、6.6.x、6.7.x...）
- 模块裁剪（exclude/unset 不需要的模块）
- 编译选项调整（优化级别、调试信息等）

### 4. 幂等设计，断点续传

- 每个构建阶段都有明确的完成标记
- 支持中断后继续构建，不会重复执行已完成步骤
- 增量编译友好，节省开发时间

## 适用场景

### 嵌入式 Linux 开发

| 目标架构 | 典型设备 |
|---------|---------|
| ARM     | 树莓派、各类 ARM 开发板 |
| ARM64   | Rockchip、NVIDIA Jetson、高通平台 |
| MIPS    | 路由器、物联网设备 |
| RISC-V  | 新兴 RISC-V 开发板 |

### 需要特定 Qt 版本的定制化编译

- 项目依赖 Qt 的某个特定版本
- 需要启用/禁用 Qt 的某些模块（如 QtWebEngine、QtMultimedia）
- 需要自定义编译选项（静态库、动态库、调试符号等）

### 规避现场编译 Qt 的复杂性

在客户现场或生产环境中编译 Qt 经常面临：
- 依赖库缺失
- 编译参数复杂
- 耗时漫长
- 环境差异导致编译失败

本项目预编译 Qt SDK，打包成 tar.xz，直接解压即可使用。

## 为什么选择本项目

### 1. 配置极简，不造轮子

| 特性 | 说明 |
|-----|------|
| **零学习成本** | 所有 configure 参数直接透传 Qt 官方 CMake，与手动编译 Qt 一致 |
| **标准化** | 完全遵循 Qt 官方编译流程，没有任何"魔法" |
| **可维护性** | Qt 版本升级只需修改配置文件中的版本号 |

### 2. 自动化程度高

```
输入：配置文件
输出：Qt SDK tar.xz + SHA256 校验和
```

中间过程全自动：
- 源码下载 → 依赖检测 → CMake 配置 → 编译 → 安装 → 打包 → 校验

### 3. 生产级质量保证

- 完整的错误处理与日志记录
- 每个构建阶段都有状态检查
- 支持并行编译加速
- 生成的 Qt SDK 可直接集成到 CI/CD 流程

### 4. 开箱即用的打包方案

编译完成后自动生成：
```
qt-6.X.X-linux-arm64.tar.xz           # Qt SDK 压缩包
qt-6.X.X-linux-arm64.tar.xz.sha256    # SHA256 校验和
```

解压即可在目标设备上使用：
```bash
tar -xf qt-6.X.X-linux-arm64.tar.xz -C /opt/
export PATH=/opt/qt-6.X.X/bin:$PATH
```

## 项目结构

```

├── build.sh           # 主构建脚本
├── cmake/             # CMake 辅助脚本
├── config/            # 配置文件目录
│   ├── qt_host.conf
│   ├── qt_target.conf
│   └── toolchain.cmake
├── scripts/           # 功能模块脚本
│   ├── download.sh    # 源码下载
│   ├── build.sh       # 编译构建
│   ├── package.sh     # 打包发布
│   └── ...
└── document/          # 项目文档
```
